    Proyecto Lombok

    Librería para condimentar Java

    1. Descripción

    Lombok es una librería que todo desarrollador Java debería incorporar a sus proyectos desde el primer momento. El objetivo principal del proyecto Lombok es el de evitar tener que escribir código repetitivo.

    Ya nunca más tendremos que escribir los método get o set, los método equals o hashcode, sobrescribir el método toString, podremos automatizar nuestras variables de tipo logging, y mucho más. Y todo esto con una sola anotación.

    Java es un gran lenguaje, pero en ocasiones es muy minucioso con el código en tareas habituales o en el cumplimiento de ciertas prácticas de su infraestructura. Estas, a menudo, no dan ningún valor añadido a al parte de negocio de nuestros programas. Es aquí donde Lombok nos facilitará nuestro trabajo y nos permitirá ser más productivos.

    Su forma de funcionamiento es la de una extensión que se conecta con el proceso de construcción y autogenera código binario Java en los archivos .class, dependiendo de una serie de anotaciones de este proyecto que iremos introduciendo en nuestro código.

    Nota

    Según la página oficial del proyecto Lombok, es una librería java que automáticamente se conecta con nuestro editor y herramientas de construcción para ponerle pimienta a nuestro Java (quizás de aquí el logotipo elegido para este proyecto).

    2. Descarga e instalación de Lombok

    Incluirlo en nuestros desarrollos es sencillo, y su página oficial dispone de instrucciones detalladas dependiendo del sistema usado.

    Me voy a centrar en una instalación común, en un proyecto Maven y en un entorno de desarrollo integrado común, en este caso, Spring Tool Suite 4 para Eclipse. Se puede consultar la página oficial para otras opciones de herramientas de construcción (Javac, Gradle, Ant) y entornos de desarrollo (Eclipse, IntelliJ IDEA, Netbeans, Visual Studio Code, Jboss, MyEclipse) o plataformas (Android, GWT).

    2.1. Descarga

    Podremos descargar la última versión desde la página oficial. Se descargara la librería con el nombre lombok.jar. En este caso la instalación la podemos hacer desde nuestra carpeta habitual de descargas.

    Este archivo también se puede descargar del repositorio central de Maven, aunque en este caso el nombre incluirá el número de versión, por ejemplo lombok-1.18.8.jar, pero se trata del mismo archivo. En este caso la instalación la podemos hacer desde nuestra carpeta habitual de Maven.

    2.2. Instalación en Spring Tool Suite 4 para Eclipse

    El proceso de instalación en Spring Tool Suite (STS) es semejante a la instalación en Eclipse u otros IDEs basado en este último, MyEclipse, Red Hat JBoss Developer Studio, JBoss Developer Studio (JBDS).

    2.2.1. Instalación en Windows

    Haciendo doble clic en el archivo lombok.jar, comenzará el instalador, que encontrará los IDEs que tengamos instalados basados en Eclipse, y nos solicitará que seleccionemos en qué entornos queremos instalarlos.

    Esta misma herramienta la podremos usar también para desinstalarlos.
